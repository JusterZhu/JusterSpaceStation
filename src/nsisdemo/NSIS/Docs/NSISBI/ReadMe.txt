NSISBI旨在消除NSIS中当前的2GB限制。此版本增加了对使用单独文件存储安装数据的支持，因此允许安装程序的最大理论大小为8EB。

使用外部文件的命令：

   OutFileMode auto | aio | data | stub。

“aio”参数与经典nsis（一体式安装程序）相同。

“data”参数强制创建外部文件，而不管其大小（零数据安装程序除外）。

“stub”参数是“数据”模式的专用版本，它将exehead转换为可下载的存根，并可以使用插件下载主安装文件。

“auto”参数是默认设置，因此它会创建高达4GB限制的一体式安装程序，超过该限制后，它会切换到外部文件以获取数据。

  NSISBI在制作一体式安装程序（无外部数据，单安装包小于4GB）时，使用固定语法实现支持固实极限压缩！（本3.08安装包即为极限压缩制作，体积已经比第一个版本小三分之一）

 详细用法见脚本范例文件夹下的“NSISBI使用固实压缩实例”，语法解释见Docs下NSISBI目录ReadMe.txt.

 还存在一个未记录的“Target”命令，其允许选择目标架构。目标cpu字符集的有效值为：x86-ansi, x86-unicode, amd64-unicode。
